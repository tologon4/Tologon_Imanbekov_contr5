
Создал базу данных contr5_database. 

База данных предназначена для учета поставок товаров в склады.

       ---------1----------

Для учета товаров создал таблицу products.
Для описания товара в таблице products есть поля: 
	1. id - Персональный номер товара 
	2. category_id - Категория товара 
	3. brand_id - Брэнд товара 
	4. product - Название товара 
Таблица категорий связана с таблицей products внешним ключем category_id
Таблица брендов связана с таблицей products внешним ключем brand_id

Для описания категорий товаров создал таблицу categories
Для описания категория в таблице есть поля:
	1. id - Персональный номер категория 
	2. category - Название категория 
	3. description - Описание категория  

Для описания брендов товаров создал таблицу brands
Для описания бренда в таблице есть поля:
	1. id - Персональный номер бренда 
	2. brand - Название брэнда 

      ---------2---------- 

Для учета поставок товаров от контрагентов в склады ,
 	. выдачи товаров со складов контрагентам, 
	. перемещения товаров из одного склада в другой, 
	. количества товаров,
	. дату действий 
создал следующие таблицы:

1. Warehouses 
2. Counterparties
3. Actions


1. Таблица warehouses предназначена для описания складов 
Для описания склада в таблице есть поля:
	1. id - Персональный номер склада
	2. warehouse - Название склада
	3. city - Город, где располагается склад

2. Таблица counterparties предназначена для описания контрагентов
Для описания контрагентов в таблице есть поля:
	1. id - персональный номер контрагента
	2. counterparty - название контрагента

3. Таблица actions предназначена для учета и описания всех действий 
Для учета всех действий в таблице есть поля:
	1. id - Персональный номер действия
	2. product_id - персональный номер продукта 
	3. counterparty_id - персональный номер контрагента
	4. from_warehouse_id - персональный номер склада откуда берется товар
	5. to_warehouse_id - персональный номер склада куда доставляется товар
	6. qty - количество товара
	7. action_date - дата действия 

Таблица продуктов связана с таблицей actions внешним ключем product_id
Таблица контрагентов связана с таблицей actions внешним ключем counterparty_id
Таблица склада откуда берется товар связана с таблицей actions внешним ключем from_warehouse_id
Таблица склада куда доставляется товар связана с таблицей actions внешним ключем to_warehouse_id


      ---------3---------

Действия в базе:
	1. При поступлении товара на склад от контрагента, в таблице actions:
		. в поле counterparty_id мы указываем персональный номер (foreign key) контрагента с таблицы counterparties
		. в поле to_warehouse_id мы указываем персональный номер (foreign key) склада куда поступил товар, с таблицы warehouses 
	2. При перемещении товара со склада на склад, в таблицу actions:
		. в поле from_warehouse_id мы указываем персональный номер (foreign key) склада откуда поступил товар, с таблицы warehouses 
		. в поле to_warehouse_id мы указываем персональный номер (foreign key) склада куда поступил товар, с таблицы warehouses 	
	3. При выдаче товара со склада контрагенту, в таблицу actions:
		. в поле counterparty_id мы указываем персональный номер (foreign key) контрагента с таблицы counterparties
		. в поле from_warehouse_id мы указываем персональный номер (foreign key) склада откуда поступил товар, с таблицы warehouses
